Abstract

KANR: Key-Frame Animator is a web app to help users get familiar with animation process.

Introduction

Our high-level goals are to:

  • Allow users to create any kind of animation.
  • Create high-level functionalities.
  • Distract users from cotidian problems.
  • Create 3d objects
  • Ability to manage high loads of data

Description

First step: design the wireframe
Next steps: think about the flow

    This application's purpose is oferring a most pleasant experience to users that are interested in creating their own animations, for personal reasons or just for fun.

    So, any user can create 2D graphical objects based on Bezier Curves. To create an object the user will be able to choose the desired number of sides for the shape and the shape willl be generated automatically. After that the user can simply shape them around as he wants. A new created and added object will have it's own layer of transparency so the user can have multiple objects and diferrent movements for his animation.

    The user will then be able to modify the animation with multiple options. He can access those options trough the menu on the right.

Objects's properties mean:

  • Size
  • Position
  • Color
  • Transparency level
  • Rotation degrees
Animation's properties mean:
  • Choosing timing function type: ease-ing / linear
  • Duration of the animation
  • Relative rotation point for the object

    By default the animation will last 5 seconds. The user can select from the progress bar a time and then apply new properties to the object so he can manipulate his animation as he desires. The current layer's play can be started/paused using the button near the progress bar. The trash can will, on click, delete the current layer from the animation. More objects can be animated by adding a new layer with a new/an existing object and applying new properties on that layer.
The user must first select the layer he wants to edit from the Layers tab on the right menu. If the user isn't sure about the animations he created, the Play Animation button from the menu will open a popup with the full animation as a preview. If the user doesn't like it he can edit the animation more and more until it's perfectly fit to his imagination then export it in a zip into his computer(with the html + css for that animation).
